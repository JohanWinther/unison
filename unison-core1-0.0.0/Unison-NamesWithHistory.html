data NamesWithHistory Source #

NamesWithHistory contains two sets of Names, One represents names which are currently assigned, the other represents names which no longer apply, perhaps they've been deleted, or the term was updated and the name points elsewhere now.

Constructors

NamesWithHistory 

Fields

  • currentNames :: Names

    currentNames represent references which are named in the current version of the namespace.

  • oldNames :: Names

    oldNames represent things which no longer have names in the current version of the codebase, but which may have previously had names. This may allow us to show more helpful context to users rather than just a hash.

Find all terms whose name has a suffix matching the provided HashQualified.

If the hash-qualified name does not include a hash, then only current names are searched. Otherwise, old names are searched, too, if searching current names produces no hits.

importing :: [(Name, Name)] -> NamesWithHistory -> NamesWithHistory Source #

Given a mapping from name to qualified name, update a Names, so for instance if the input has [(Some, Optional.Some)], and Some is a constructor in the input Names, the alias Some will map to that same constructor and shadow anything else that is currently called Some.

Only affects currentNames.
